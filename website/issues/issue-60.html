S4TF News - Issue 60, May 3rd 2020

Welcome to the Issue #60 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • The 1st release candidate for Swift for TensorFlow 0.9.0 is now available. Grab a copy from the Release Candidate's section from the installation instructions. The S4TF v0.9 Release Notes document guides you through all changes since v0.8, new changes and known issues. Please give this release a try and report any issues you find on JIRA or GitHub issues on Swift-APIs.

  • On Friday 1st of May, the 30th SIG Swift Design meeting took place. Here you have the highlights:

    • Anthony Platanios presented his work on Jelly Bean World a framework for never ending learning using Swift for TensorFlow.

    You can find the notes and all questions raised in this google doc.

  • Wojciech Czarnowski shared his project Language2motion, which has as goal to create multi-modal implementation of Transformer architecture in Swift. Check out the announcement here.

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-31399: Lowers PythonKit deployment target to macOS 10.10.

  • PR-31368: Disables integrated and LLDB REPL for the Swift for TensorFlow toolchain.

Swift TensorFlow APIs

  • PR-920: Renames _lazyTensor to _lazyTensorHandle. Resolves TF-701.

  • PR-915: Updates .swift-format according to configuration from the swift-5.2-branch of swift-format.

  • PR-917: Makes PrintX10Metrics available.

  • PR-910: Converts func nonuniformInferenceBatches into a type: NonuniformInferenceBatches.

  • PR-864: Introduces Epochs: a new API that aim to replace the current Dataset in swift-apis and Batcher in swift-models.

  • PR-907: Adds ability to log new graph and shape changes.

Swift Models

  • PR-427: Introduces ShuffleNetV2.

  • PR-406: Adds example on Recommendation using Neural Collaborative Filtering . Resolves Issue-151.

  • PR-464: Specifies devices explicitly in BERT-CoLA.

  • PR-401: Introduces 1D VariationalAutoencoder.

  • PR-468: Adds eager / X10 backend selection to the benchmarks.
