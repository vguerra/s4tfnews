S4TF News - Issue 47, February 2nd 2020

Welcome to the Issue #47 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-29590: Speeds up compile time by 3.5x by caching derivative function types. Resolves TF-1133.

  • PR-29564: Fixes undiagnosed active modify accessor applications. . Resolves TF-1115.

  • PR-29541: Adds SILFunctionType method assertion.

  • PR-29543: Makes @noDerivative attribute imply non-varying semantics. Now, @noDerivative can be declared on all function-like declarations: func, var, init, and subscript.

Swift TensorFlow APIs

  • PR-657: Conforms all RNN cell State structs to the same protocols.

  • PR-653: Adds documentation to Sequential.

  • PR-651: Adds support for installing the package by extending the CMake build system.

  • PR-650: Removes bad api usage in eager runtime.

Swift Models

  • PR-292: Fix the shape of the last dense layer in Resnet18 and ResNet34.

  • PR-287: Minor tweaks to the Gym models to allow for easier import internally.
