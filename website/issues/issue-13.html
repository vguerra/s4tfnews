S4TF News - Issue 13, June 9th 2019

Welcome to the Issue #13 of Swift for Tensorflow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for Tensorflow project.

Enjoy!

News and Articles

  • Progress on Google Summer of Code 2019 projects:

    • Ayush Agrawal is working on a set of tutorials for Swift for Tensorflow that target beginner and intermediate users. You can find the notebooks he's been working on GitHub. As well, he's sharing the evolution of his project on Medium in case you want to follow closely his progress. (original post).

    • Karthik Ramesh Iyer's project is to develop a data visualization library in Swift. His library is called SwiftPlot. (original post).

  • Rahul Bhalley has shared his implementation of Wasserstein GAN in Swift for Tensorflow. Check out his article on Medium and the code on GitHub.

  • Paige Bailey is now a Product Manager for TensorFlow focusing, among other topics, on Swift for TensorFlow.

  • A new Xcode 10 Development snapshot is now available for you to try out Swift for Tensorflow. Get it here.

S4TF Commits

Swift Compiler (tensorflow branch)

  • Partial fixes on memory leaks:

    • PR-25300: Adjoint memory leak.
    • PR-25294: Control flow automatic differentiation memory leak.

  • PR-25268: Optimizes adjoint buffers for address projections as they become projections into their adjoint base buffer.

  • Adding linear argument to attribues @differentiable and @differentiating:

    • PR-25269: Reflects whether functions are marked as linear or not. Serializes/Desrializes the argument.
    • PR-25228, PR-25257: Add ability to handle linear argument for @differentiable attribute when parsing.

  • PR-25273: Enables control flow automatic-differentiation by default.

  • PR-25255: Unifies block generation logic as handling entry adjoint block generation in automatic-differentiation for control flow does not differ much from the general one.

  • PR-25057: Adds support for differentiation of conditionals, allowing to differentiate nested conditionals, recursion and var allocations of Tuples and Structs.

  • PR-25235: Following decisions on naming conventions for SE-0253 this replaces func call with func callAsFunction.

  • PR-25239: Removes all TensorFlow related tests as they have been migrated to swift-apis repository.

  • PR-25238: Removes TensorGroup tests for derived conformances as they have been moved already to swift-apis repository.

Swift TensorFlow APIs

  • Work on Hinge losses:

    • PR-187: Adding Squared Hinge Loss.
    • PR-188: Adding Categorical Hinge Loss function.
    • PR-185: Adding Hinge Loss function.

  • PR-182: Adds meanAbsoluteError.

  • PR-173: Introduces LazyTensor and LazyTensorOperation types.

  • PR-168: Refactors and renames convolution and poolong operators to improve discoverability.

  • PR-176: Converts copy(of:) to an extention method of TensorHandle named makeCopy() to avoid function lookup errors.

  • PR-172: Following decisions on naming conventions for SE-0253 this replaces func call with func callAsFunction.

  • PR-169: Enables compatibility with TPUs by changing seed type from Int64 to Int32.

  • PR-165: Introduces new requirements to TensorArrayProtocol and TensorGroup to use _AnyTensorHandle.
