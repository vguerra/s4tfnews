S4TF Commits

Swift Compiler (tensorflow branch)

  • : Adds support for differentiating Array literals initialization.

  • PR-25951: Renames AdjointEmitter class to PullbackEmitter.

  • PR-25684: Adds type-checking for top-level functions and methods annotated with @transposing.

  • PR-25924: Fixes compiler crash when differentiating a function that invokes a differentiable initializer.

  • PR-25914: Introduces support for Differentiable conformance derivation for class types.

Swift TensorFlow APIs

  • Bug fixes for _vjpConv2DBackpropInput:

    • PR-333: Fixes error on Incompatible shapes.

    • PR-331: Fixes bug of incorrectly passing the shape argument to conv2DBackpropFilter as the filterSizes argument instead of passing the filter shape,

  • Improvements to random seed generation:

    • PR-328: Improves generation of default random seeds when in multi-process settings.
    • PR-320.

  • PR-327: Pairs Tensor.batchGathering(atIndices:) implementation with it's Python counterpart, supporting variable number of batch dimensions and an axis to gather along.

  • PR-324: Teaches Dense Layer to use the batch variant of matmul when rank of weight is equal 3.

  • PR-326: Allows composable Layers by introducing Sequential Layer.

  • PR-322: Makes all Layer's tangent vector conform to VectorProtocol & ElementaryFunctions & PointwiseMultiplicative. Introduces as well ParameterLessLayer protocol.

  • PR-323: Renames LazyTensor to LazyTensorHandle.

  • PR-319: Standarizes initialization of Layer's parameters introducing typealias's: ParameterInitialier and Activation.

  • PR-321: Improves documentation for convolutional Layers.

  • PR-312: Introduces orthogonal initializer for Tensors.
