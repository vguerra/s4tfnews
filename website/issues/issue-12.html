S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-25151: Fixes issues on user-defined derivatives for stored properties. As consequence user-defined derivatives for stored properties are no allowed and a name-correpondence is required between fields in T.TangentVector and T. Kudos to Bart Chrzaszcz for this great PR write-up.

  • PR-25176: Refactors code that checks for satisfaction of generic differentiation requirements.

  • PR-24452: Moves out the module Tensorflow into the swift-apis repository.

  • PR-24708: Fixes bug where SourceKit would not work with .swiftinterface files.

  • PR-25040: Puts in place checks to make sure PythonLibrary.useVersion is not called more than once.

Swift TensorFlow APIs

  • PR-158: Moves in all tests for TensorGroup from swift repository.

  • PR-159: Introduces the Layers: GlobalMaxPool1D, GlobalMaxPool2D and GlobalMaxPool3D.

  • PR-156: Introduces Tensor.gathering.

  • Adding support for the following operations:

    • PR-152: tile.
    • PR-146: expm1.
    • PR-145: log1p.
    • PR-144: sign.

  • PR-141: Speeds up Tensor shape/rank operators by using TensorFlow Eager API.

  • PR-143: Refactors matmul operation so that it uses internally Raw.batchMatMulV2.

  • PR-109: Moves in the module Tensorflow from the swift repository.

  • PR-102: Brakes all code in Layer.swift into different files within a Layers directory.
