S4TF News - Issue 56, April 5th 2020

Welcome to the Issue #56 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • On Friday 3rd of April, the 28th SIG Swift Design meeting took place. Here you have the highlights:

    • Alex Suhan made a presentation on how Swift for TensorFlow runs on XLA devices (TPUs and GPUs). He took us on a deep dive on X10, which is an implementation of S4TF Tensor operations using XLA. His slides are available here.

    Recording from this session is available online.

    As well, you can find the notes and all questions raised in this google doc.

  • You can now find Development Snapshots of S4TF with X10 support at the Installation page.

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-30726: Delets tests that have been upstreamed.

Swift TensorFlow APIs

  • PR-804: Fixes gradient values used in Optimizer tests to match the ones in reference in Python. Resolves Issue-759.

  • PR-800: Adds option DEFAULT_BACKEND_EAGER to allow different default backends.

  • PR-795: Marks Dataset APIs as deprecated.

  • PR-735: Adds support for x10 in CI.

  • PR-713: Removes Python module compatibility.

Swift Models

  • PR-409: Introduces MobileNet v3 ( MobileNetV3Large and MobileNetV3Small).

  • PR-431: Adds ResNetCIFAR10 benchmark.

  • PR-430: Changes default download location for BostonHousing and CIFAR-10 datasets.

  • PR-428: Removes device placement CIFAR10 and MNIST examples, as they are already loaded on the device.

  • PR-408: Introduces MobileNetV1-Imagenette and MobileNetV2-Imagenette targets.
