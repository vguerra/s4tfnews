S4TF News - Issue 21, August 4th 2019

Welcome to the Issue #21 of Swift for Tensorflow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for Tensorflow project.

Enjoy!

News and Articles

  • On Friday 2nd of August, the 5th SIG Swift Design meeting took place. Here you have the highlights:

    • Dan Zheng presented the internals of how the Automatic Differentiation is implemented in the Swift Compiler. If you missed this presentation I encourage you to have a read at the Implementation Overview document that Dan has put together.

    You can find the meeting notes and all questions raised in this google doc.

The mailing list

  • Cristian Garcia shared on the mailing list NDArray, influenced by TensorFlow's Tensor API and numpy documentation. One of the key goals of the library is to make NDArray and its operations usable within Swift for Tensorflow. (original message).

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-26426: Fixes compiler crash when differentiating generic functions with non-Differentiable parameters. Resolves TF-691 and TF-697.

  • PR-26407: Checks and procudes diagnose for usage of non-differentiable original function in arguments/results. Resolves TF-687

  • PR-26404: Fixes compiler crash during generic curry thunk cloning. Resolves TF-688.

  • PR-26405: Fix compiler crash during @differentiable attribute's type checking. Resolves TF-521.

  • PR-26402: Improves diagnostics for @differentiable's attributes requirements that are not met.

  • PR-26380: Makes sure to always thunk JVPs/VJPs during SIL generation. Resolves: TF-546, TF-667, TF-682.

  • PR-26389: Makes use of ASTMangler to mangle generated AD functions: JVPs, VJPs, diffentials and pullbacks.

  • PR-26384: Fixes memory leak produced during emission of subset parameters thunks.

Swift TensorFlow APIs

  • PR-129: Introduces an experimental implementation of Complex Numbers based on Xiaodi Wu's NumericAnnex implementation.

  • PR-409: Cleans up old tracing implementation.

  • PR-406: Materializes inputs simultaneously when executing 0-output LazyTensorOperation. Resolves TF-604.

  • PR-405: Exposes LazyTensorOperation of a LazyTensorHandle.

  • PR-401: Moves computation of rank and shape properties to TFETensorHandle.

  • PR-398: Makes publicly readable the list of available devices on the host.

  • PR-385: Refactors tracing utilities in terms of LazyTensor.

Swift Models

  • PR-185: Extracts common CIFAR-10 and ResNet code.

  • PR-182: Extracts common MNIST dataset code and introduces an example that uses classic LeNet-5 model.
