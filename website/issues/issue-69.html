S4TF News - Issue 69, July 5th 2020

Welcome to the Issue #69 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

S4TF Commits

Swift Compiler (master branch)

  • PR-32714: Disallows differentiation of opaque-result-typed functions. Resolves SR-12656.

  • PR-32670: Fixes unexpected non-differentiable property access error. Resolves SR-13134.

  • PR-32614: Registers derivative for the Get and Set Accessors. Partially resolves TF-1193.

  • PR-32639: Fixes protocol witness SILGen for @differentiable class methods. Resolves TF-1180.

  • PR-32629: Adds support for differentiation of functions with multiple results in SIL. Resolves TF-983.

Swift TensorFlow APIs

  • PR-1035 and PR-1034: Fix ParameterlessLayer conformances given that Associated type inference behavior was changed in PR-32578.

  • PR-1033: Adds missing StringTensor initializer call: self.init(handle: handle).

  • PR-1023: StringTensor updates based on upcoming changes to tensorflow master branch: Eliminate unnecessary packing/unpacking of tstring tensors across the C-API.

Swift Models

  • PR-612: Ports benchmarks to swift-benchmark.

  • PR-620: Fixes ParameterlessLayer conformances given that Associated type inference behavior was changed in PR-32578.

  • PR-618: Restructures checkpoint files, adding an initial filesystem abstraction, into a Checkpoints module.
