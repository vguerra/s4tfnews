S4TF News - Issue 32, October 20th 2019

Welcome to the Issue #32 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • Ted Kremenek expressed on the Swift forums, on behalf of the Core Team, their interest in evaluating incorporating AD capabilities directly into the language. The #S4TF team has already put together a plan to upstream AD features from #tensorflow branch to #master.

  • Mohd Sanad Zaki Rizvi has written A Comprehensive Guide to Learn Swift from Scratch for Data Science. He writes about Swift basics and the feature of the language in the field of Data Science.

  • The last September 18th, Olzhas Akpambetov, gave a talk at TensorFlow London meetup on Swift for TensorFlow. Watch the recording of the talk on Youtube.

  • On Friday 18th of October, the 13th SIG Swift Design meeting took place. Here you have the highlights:

    • Alex Suhan talked about Swift binary size reductions.

    • Brennan Saeta presented the design proposal docuemnt on Layering Swift APIs.

    You can find the meeting notes and all questions raised in this google doc.

  • Our @s4tfnews twitter account has surpassed 500 hundred followers. Thank you all for your likes, shares and re-tweets.

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-27596: Adds --enable-experimental-differentiable-programming build-script flag.

  • PR-27754: Marks derivative function thunks as always inline. Resolves TF-925.

  • Refactoring of tests to use Tracked<Float> in AD Tests to have leak checking:

    • PR-27767.
    • PR-27733.

  • PR-27752: Makes sure that derivative function thunks are not transparent.

  • PR-27719: Adds SIL instruction differentiability_witness_function. Resolves TF-915.

  • PR-27720: Adds cloned curry thunks to generated function list. Resolves TF-918.

  • PR-27652: Teaches SILGen to generate differentiability witnesses from @differentiable/@differentiating attributes. Resolves TF-869.

  • PR-27711: Makes sure that all pullback indirect results are destroyed after adjoint accumulation. Resolves TF-914.

  • PR-27689: Aligns differentiable_function instruction's syntax with that of linear_function. Resolves TF-909.

  • PR-27688: Renames DifferentiableFunctionExtractee to NormalDifferentiableFunctionTypeComponent and makes it a top-level type. Resolves TF-904.

  • PR-27687: Introduces support for function type conversions of @differentiable(linear). Resolves TF-900.

  • PR-27684: Add diagnostic for unsupported forward-mode differentiation of functions with multiple basic blocks.

  • PR-27637: Introduces SIL instructions: linear_function and linear_function_extract used for creation/destruction of @differentiable(linear) functions. Resolves TF-907.

  • PR-27661: Adds support for lowering function types @differentiable(linear) to LLVM IR. Resolves TF-902.

Swift TensorFlow APIs

  • PR-529: Adds support for TensorFlow checkpoint file reader.

Swift Models

  • PR-218: Enables masking on VJP for causallyMasked (_vjpCausallyMasked). Resolves Issue-217

  • PR-215: Enables downloading MNIST Dataset from a remote URL. Partially solves Issue-206.
