S4TF News - Issue 23, August 18th 2019

Welcome to the Issue #23 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • On Friday 16th of August, the 7th SIG Swift Design meeting took place. Here you have the highlights:

    • Bart Chrzaszcz made a deep dive into the details of the Forward Mode Compiler Transform.

    • Brennan Saeta went over the list of items the S4TF core team is focusing in the short-term:

      • Automatic differentation.
      • Metaprogramming with quasiquotes and MLIR.
      • C++ interoperability.
      • TPU Support.
      • Developper tooling.

    You can find the meeting notes and all questions raised in this google doc.

  • Choong Ng has put together a nice guide for you to get started with Swift for TensorFlow with Docker on macOS. It guides you through the necessary steps to have the latest release of S4TF ( v.0.4.0) working on Jupyter notebooks, making it easy for you to start exploring fast.ai S4TF Notebooks.

  • As part of his fast.ai journey, Rick Wierenga, has written two lesson summaries:

    • Lesson 13 - Linear Regression in Swift for Tensorflow which show cases how to get started with Swift for TensorFlow.
    • Lesson 14 - An end at the beginning that explains how Rick re-wrote the project of the very 1st Lesson, A fruit classifier, using Swift for TensoFlow. He even created a Swift package to access Kaggle API: SwiftKaggle.

  • Karthik Ramesh Iyer shared the progress of his GSoC 2019, a Data Visualization Library for Swift, talking about how he leveraged the power of Swift Generics and C interoperatiblity in his library.

  • Paige Bailey has shared on Twitter a link to the Swift Development Environment VSCode Extention. If you like to code in Visual Studio defenitely check it out. (Github repo).

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-26157: Moves the Differentiation pass in between DefiniteInitialization and OwnershipModelEliminator passes. Resolves TF-709 and TF-585.

  • PR-26659: Cleans up the temporary TFE_InferShapes_Transition API.

  • PR-26587: Renames PullbackInfo to LinearMapInfo to better represent what it stores (info about JVP/VJP generation).

Swift TensorFlow APIs

  • PR-457: Adds derivative tests for DenseLayer. Partially resolves Issue-402.

  • PR-454: Refactors TensorBuffer.create(create:withInitializer:) to use appropiate Array initializer. Resolves Issue-452.

  • PR-453: Adds handling of inputTensors during shape inference.

  • PR-446: Adds support for tensorFunctionPointer in LazyTensor.

  • PR-445: Cleans up references to TFE_TensorHandleIsConcrete.

  • PR-440: Fixes tests on LayerNorm.

  • PR-439: Makes sure order of output is preserved when explicit traces are generated.

  • PR-422: Adds overloaded functions for losses that have a reduction parameter. Resolves Issue-422.

Swift Models

  • PR-194: Removes usage of deprecated allDifferentiableVariables.

  • PR-193: Adds fixes on implementation of WideResNet.

Swift for TensorFlow Docs

  • PR-225: Introduces an experimental prototype for Swift quasiquotes.

  • PR-227: Introduces a parser for SIL.
