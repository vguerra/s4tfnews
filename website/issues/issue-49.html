S4TF News - Issue 49, February 16th 2020

Welcome to the Issue #49 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • The 2nd release candidate for Swift for TensorFlow 0.7.0 is now available. Grab a copy from the Release Candidate's section from the installation instructions. The S4TF v0.7 Release Notes document guides you through all changes since v0.6. Please give this release a try and report any issues on JIRA you find.

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-29800: Fixes compilerr crashes related to generic specialization of partial_apply operands to differentiable_function instructions. Resolves TF-891 and TF-1126.

  • PR-29757: Adds support for @differentiable class initializers. Resolves SR-12151 and SR-12153.

  • PR-29758: Cleans up SILFunctionType::getAutoDiffTransposeFunctionType

  • PR-29749: Adds support for ref_element_addr differrentiation. Resolves SR-12152.

  • PR-29129: Adds negative tests for derivate registration of an @_alwaysEmitIntoClient function. Ref: TF-1103.

Swift TensorFlow APIs

  • PR-678: Removes operators that were not accessible outside of swit-apis and rewrite the optimizer code using .scaled(by:) and .adding().

  • PR-664: Updates the Adam optimizer class API docs.

  • PR-667: Reverts preconditions added to reshaped in PR-557.

Swift Jupyter

  • PR-89: Prepares for coming changes in toolchain regarding location of lldb python lib.

Swift Models

  • PR-322, PR-312 and PR-304: Use explicitly typed math operations to enable Windows builds.

  • PR-321: Reorganizes the BERT model, splitting out the CoLA dataset and a BERT-CoLA training example.

  • PR-231: Adds support for BERT.

  • PR-307: Adds executable for Transformers.

  • PR-301: Adds executable for VGG-Imagewoof.

  • PR-277: Introduces checkpoint reader for TensorFlow v2 checkpoint files.

  • PR-303: Introduces a CMake based build.
