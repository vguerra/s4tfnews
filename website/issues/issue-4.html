S4TF News - Issue 4, April 7th 2019

Welcome to the Issue #4 of Swift for Tensorflow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for Tensorflow project.

Enjoy!

News and Articles

  • The Google Brain team at Google is hiring Swift developers with Machine Learning skills. If you are interested then check out the Job posting shared by Chris Lattner on twitter.

The mailing list

  • Anthony Platanios asked for advice on how to include a CMake target in SPM. His goal was to add support for Gym retro to Swift. Check out his resulting implementation: Retro Games in Gym for Swift. (original message).
  • Following up a discussion Anthony Platanios initiated last week about fancy ways of indexing Tensors, he started a discussion on the Swift forums about the same topic: Advanced Arrat/Tensor Indexing.
  • Shawn Morel raised the question on the mailing list about what is the current state on how to track upstream changes into the Swift compiler. Following up some of the points raised, Richard Wei asked on the Swift forums, what would be a good strategy to upstream the Python module. (original message).

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-23833: Introduces usage of raw operations:

    • concatV2 for concatenating tensors w/out slow slice assignment.
    • stridedSlice to extract a strided slice of a tensor without custom computation of tensor indexes.

  • PR-23738: Adds derivatives for addition addition and substraction on AnyDerivative.

  • PR-23719: Fixes a bug that would crash the compiler if differentiation properties have no getter.

Swift TensorFlow APIs

  • PR-80: Initial work towards generalization of RNNs. It introduces a representation of inputs for RNNs (RNNInput) and the protocol RNNCell to represent a RNN cell.
  • PR-73: Adds support for explicit paddings to convolutional Ops.
  • PR-72: Introduces GlobalAveragePooling3D.
  • PR-66: Introduces GlobalAveragePooling1D.

Swift Models

  • PR-86: Introduces pre-activated Residual Networks. Variants PreActivatedResNet18 and PreActivatedResNet34.
  • The following variants for Residual Networks were added:
    • PR-76: Adds ResNet20, ResNet32, ResNet44 and ResNet56.
    • PR-81: Adds ResNet101 and ResNet152.
    • PR-79: Adds ResNet18 and ResNet34.

Swift for Tensorflow Docs

  • PR-165: Adds a Jupyter notebook explaining how Python Interoperability works.

Starter bugs

Looking to contribute to the project? Have a look at the currently tickets for starters:

  • TF-419: Improve tensor printing.
  • TF-416: Produce error upon second usage of PythonLibrary.useVersion.
  • TF-130: Add API to serialize a @convention(tensorflow) function as a graph.
  • TF-67: BumpPtrAllocating some classes that have SmallBitVector fields.
