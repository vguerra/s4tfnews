S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-24195: Improves install instructions with right version needed of Bazel.

  • PR-24191: Tensor.expandingShape supports [Int] or Int like its counterpart Tensor.squeezingShape does.

  • PR-24164: Fix throwing errors in backward pass for sum(squeezinAxes:) and mean(squeezinAxes:).

  • PR-24120: Adds split Op for tensor.

  • PR-24122: Introduces first implementation of evolution proposal SE-0253 for callables. (This is actually a 2nd version, if you want to have a look at the first implementation that got reverted check out PR-24086).

  • PR-24096: Adds splitV which splits a Tensor along one dimention.

  • PR-23684: Introduces advanced Indexing and Slicing for Tensors.

  • PR-23183: Makes arrays differentiable.

  • PR-23777: Adds _TFCExecuteOp which dispatches an eager op returning a Tensor group. As well, introduces raw operations: saveV2 and restoreV2 and AnyTensor protocol.

  • The following PR's make APIs use Int instead of Int32 or Int64:

    • PR-24012: Makes APIs use Int instead of Int32.
    • PR-24110: Dataset API uses Int instead of Int64.

  • PR-23891: Introduces ConvertibleFromPython protocol which defines conversion from PythonObject.

  • PR-24042: Standarizes numeric reduction Ops to have squeezing, along and nullary versions.

Swift TensorFlow APIs

  • PR-70: Introduces following improments to APIs:

    • Splits Operations into several files.
    • Adds the following initializers for Tensor: Tensor(stacking:alongAxis:), Tensor(concatenating:alongAxis:), and Tensor(tiling:multiples:) and Tensor(rangeFrom:to:stride:).
    • Introduces following Operations: Tensor.unstack(), Tensor.split(numSplits:alongAxis:), Tensor.split(sizes:alongAxis:), sigmoid(_:).

  • PR-105: Makes LSTMCell and SimpleRNNCell conform to VectorNumeric. As well, introduces a wrapper layer RNN to this cells: SimpleRNN and LSTM respectively.

  • PR-96: Introduces usage of call func ( as implemented by SE-0253). With this change you don't have to use anymore the applied method on Layers, instead you do: layer(input).

  • PR-97: Introduces the following Recurrent Layers: SimpleRNNCell, LSTMCell.

  • PR-95: Fixes array out-of-bounds errors you would get occasionally when running Optimizers. This removes as well the scalarType parameter in initializers. As well, allows experimentation by exposing some properties in optimizers: Adam, RMSProp, SGD.

  • PR-90: Renames GlobalAveragePooling to GlobalAvgPool.

  • PR-84: Makes APIs use Int instead of Int32.

  • Improvements on Tests:

    • PR-82: Unit tests for GlobalAvgPool1D, GlobalAvgPool2D and GlobalAvgPool3D.

    • PR-89: Unit tests for Flatten, Reshape layers.

Swift Models

  • PR-108: Removes Python code from MNIST example.

  • PR-89: Adds wide ResNet 16 and 28 variants.
