S4TF News - Issue 71, July 19th 2020

Welcome to the Issue #71 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • On Friday 17th of July, the 36th SIG Swift Design meeting took place. Here you have the highlights:

    • Andrey Volodin presented Sardonyx: a CLI tool that converts your ONNX model into Swift code + structured data blob, making it easy for you to reuse models that were created using PyTorch or TensorFlow.

    Recording from this session is available online) as well.

    You can find the notes and all questions raised in this google doc.

  • New Development snapshots are available for you to try out Swift for Tensorflow: Xcode 11.

S4TF Commits

  • PR-32939: Upstreams @differentiable protocol requirement tests from tensorflow branch.

  • PR-32747: Registers VJPs for SIMD subscript(_: Int) setters.

  • PR-32854: Registers JVPs for SIMD operations.

  • PR-32839: Serializes and prints the optional accessor kind in @derivative and @transpose attributes. Resolves TF-1293.

  • PR-32840: Registers JVPs for Array operations.

Swift TensorFlow APIs

  • PR-1044: Makes loss functions differentiable w.r.t. predicted and expected values.

  • PR-1043: Migrates unordered_map to Abseils node_hash_map.

Swift Models

  • PR-635: Adds datasetFilePath argument for benchmarks.
