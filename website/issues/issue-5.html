S4TF News - Issue 5, April 14th 2019

Welcome to the Issue #5 of Swift for Tensorflow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for Tensorflow project.

Enjoy!

News and Articles

  • Tatiana Shpeisman and Chris Lattner gave a Keynote at the 2019's European LLVM Developers meeting where they introduced MLIR: A Multi-Level Intermediate Representation for Compiler Infrastructure. This project is important for the Swift For Tensorflow as it would bring benefits in many areas. You can find the slides of the presentation here.

  • As you might know already, TensorFlow is an official mentoring organization for 2019 edition of GSoC. Within the list of projects proposed there are some exciting ideas for Swift for TensorFlow. The deadline for the application was the past 9th of April. Accepted projects will be announced on May 6. Stay tunned.

The mailing list

  • Anthony Platanios raised the question of when to use ShapedArray vs CPU Tensors. (original message).
  • If you, as Dough Friedman, ever wondered how all operations are defined in the Raw namespace, check out the swift-bindings repo to see out how it all works. (original message).

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-24013: Add derivative on AnyDerivative's initializer.

  • PR-23837: Improves pretty-printing of Tensors.

  • PR-23854: Enables propagation of @nondiff attribute from AST to SIL.

  • PR-23762: TensorShape now conforms to PythonConvertible.

  • PR-23811: Extends Tensor with variance method to compute it's variance along a specific axis.

  • PR-23868: Adds API for function tracing with XLA compilation.

  • PR-23763: Extends Tensor with standardDeviation method to compute the standard deviation along a specific axis. As well it adds derivaties for sum and mean along a specific axis.

  • PR-23843: Introduces usage of allSatisfy and removes custom forAll extention on Sequence.

Swift TensorFlow APIs

  • PR-87: context argument has been removed from applied method on all Layers. Instead, the newly introduced ContextManager class handles a stack of contexts for a given thread.

  • PR-86: Improvements on the RNN Cell abstraction. Introduces RNNCellInput and RNNCellOutput.

  • PR-81: Allos learning rates to be set dynamically.

Starter bugs

Looking to contribute to the project? Have a look at the currently open tickets for starters:

  • TF-416: Produce error upon second usage of PythonLibrary.useVersion.
  • TF-130: Add API to serialize a @convention(tensorflow) function as a graph.
  • TF-67: BumpPtrAllocating some classes that have SmallBitVector fields.
