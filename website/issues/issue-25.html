S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-26848: Introduces InMemoryFrontend lib, which runs compilation and emits resulting module to a memory buffer, w/out writting to the file system.

  • PR-26827: Adds zeroTangentVector to Differentiable protocol. Note: Derivation for this property is not yet implemented. Resolving partially Partially resolves TF-708.

  • Work on new protocol EuclideanDifferentiable protocol, which you can use for representing common differentiable data structures that contain both differentiable vector properties and other stored properties that do not have a derivative:

    • PR-26900: Fixes crash on derived conformances. Resolves TF-783.
    • PR-26890: Derives differentiableVectorView property from member's vector views. Resolves TF-785.
    • PR-26867: Derives conformances to the protocol. Resolves TF-777.
    • PR-26827: Introduces EuclideanDifferentiable protocol.

  • PR-26826: Makes force-unwrapping differentiable. Resolves TF-455.

Swift TensorFlow APIs

  • PR-397: Fixes order of elements in returned tuple in VJP of conv2DBackpropFilter (_vjpConv2DBackpropFilter) as differentiation is w.r.t. (x, input).

  • PR-476: Allows to automatically promote constants to inputs based on history in LazyTensor.

  • PR-473: Adds Tensor.init (linearSpaceFrom:to:count:) initializer with Scalar and Tensor parameter variants.

  • PR-497: Removes backwards compatible APIs needed for 0.4.0 release. Resolves Issue-495.

  • PR-498: Fixes Inputs for Layers ZeroPadding1D, ZeroPadding2D and ZeroPadding3D as they should have batch and feature dimensions. Resolves Issue-494.

  • PR-491: Makes Module conform EuclideanDifferentiable as well as all Layer combinators. Resolves Issue-456.

  • PR-480: Matches gradient computation for min(_:_:) and max(_:_:) to their Python's Tensorflow counterparts.

  • Adding derivate tests for following layers:

    • PR-493: ZeroPadding1D, ZeroPadding2D and ZeroPadding3D.

    • PR-493: Embedding.

    • PR-472: AvgPool.

    Partially solving Issue-402.

Swift Models

  • PR-204: Adds SqueezeNet version 1.1 (SqueezeNetV1_1) and renames prev. implementation to SqueezeNetV1_0.

  • PR-198: Adds shape-based inference tests for all image classification models.

  • PR-203: Converts Autoencoder and Catch to use Sequential. Partially solving [Issue-202]((https://github.com/tensorflow/swift-models/issues/202).

  • PR-200: Converts MNIST classifier (LeNet-5) to use Sequential.
