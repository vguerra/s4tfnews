S4TF News - Issue 51, March 1st 2020

Welcome to the Issue #51 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • On Friday 28th of February, the 24th SIG Swift Design meeting took place. Here you have the highlights:

    • Ewa Matejska leaded the discussion around S4TF project organization tasks, like release process and bug filing, as well as ideas on improving project communication. Checkout the presentation slides.
    • Dan Zheng initiated the discussion around PR-693 on swift-apis regarding renaming of Recurrent Layers.

    You can find the meeting recording, notes and all questions raised in this google doc.

  • The Agenda for this year's edition of TensorFlow Dev Summit has been released and there will be a session on Swift for TensorFlow: Swift for Tensorflow Deep Dive. Be sure to attend it if you happen to attend the event or follow the livestream on their YouTube channel.

  • A new Xcode 11 Development snapshot is now available for you to try out Swift for Tensorflow. Get it here.

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-30122: Specifies a compiler target for PythonKit on macOS.

  • PR-30120: Specifies a compiler target for tensorflow-swift-apis on macOS.

  • PR-30108: Removes deprecated method-style differential operators.

  • PR-30043: Makes sure PythonKit is installed.

  • PR-29771: Relaxes @differentiable requirement for protocol witnesses so that only public witnesses of @differentiable protocol requirements are required to have the same attribute (or one with superset parameter indices) . Resolves TF-1117:

  • PR-30013: Support inout argument differentiation (reverse-mode only). Resolves TF-1165.

Swift TensorFlow APIs

  • PR-704: Introduces more device and precision apis.

  • PR-703: Makes sure that module is installed into the macOS style paths which are honoured on both macOS and non-macOS targets.

  • PR-698: Adds tests for Optimizers. Resolves Issue-696.

  • PR-672: Introduces transposedConv2D.

  • PR-641: Fixs crash when computing gradient of tensor.mean with negative axis.

  • PR-659: Fixes Tensor.batchGathering to properly handle negative axis arguments.

Swift Models

  • PR-338: Introduces Batcher examples, specifically, showing how to assemble a standard ImageNet preprocessing pipeline on Imagenette.

  • PR-346: Adds a simple UI wrapper for GPT-2.

  • PR-345: Introduces BijectiveDictionary: A basic bi-directional mapping utility.

  • PR-352: Adds support for tests to CMake to enable running the test suite on Windows.

  • PR-331: Replaces in Transformer Python with Swift objects.

  • PR-348: Adds zero init for last BatchNorm layers in ResNet V2.

  • PR-347: Adds support for building against PythonKit.
