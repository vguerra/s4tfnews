S4TF News - Issue 19, July 21st 2019

Welcome to the Issue #19 of Swift for Tensorflow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for Tensorflow project.

Enjoy!

News and Articles

  • Jeremy Howard has written SwiftBPE which is a Swift wrapper for the C++ implementation of Neural Machine Translation of Rare Words with Subword Units ( called fastBPE.

The mailing list

  • Slawek Smyl started a discussion asking whether Swift for Tensorflow would allow multithreaded learning. (original message).

  • Cristian Garcia has created a data processing library written in Swift, called Stream, that you can use during pre-processing of your data. (original message).

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-26209: Refactors memory managenment during differentiation, having the nice side effect of eliminating memory leaks that have been reported through various channles. Resolves TF-621.

  • PR-26242: Fixes memory leak on closure capturing. Resolves TF-550.

  • PR-26232: Exposes forward mode Automatic Differentiation functions.

  • PR-26226: Improves leak-checking logic within testWithLeakChecking.

  • PR-26211: Cleans up zeros emitted when differentiating a store.

  • PR-26202: Fixes bug during parsing of SIL [differentiable] attributes. Resolving TF-656.

  • PR-26179: Improves @differentiable attribute diagnostic messages and attribute printing.

  • PR-26173, PR-26183: Rename adjoint function to pullback in the code base.

Swift TensorFlow APIs

  • PR-378: Marks all parameters of VJP's initializers as __owned as workaround for TF-667.

  • PR-375: Replaces usage of TFE_OpAddInputList by multiple calls to TFE_OpAddInput due to bogus implementation of the former.

  • PR-372: Introduces LazyTensorOperationsTracker which tracks instances of LazyTensorOperation.

  • PR-370: Extracts DeviceScopes out into it's own structure.

  • PR-366, PR-368, PR-69: Make pow(_:_:), root(_:_:) and all Tensor functions conforming to ElemntaryFunctions @differentiable.

  • PR-348: Introduces Tensor.isAlmostEqual(to:tolerance:) and renames Tensor.elementsApproximatelyEqual(_:tolerance:) to Tensor.elementsAlmostEqual(_:tolerance:).

  • PR-361: Introduces Tensor.clipped(min:max:).
