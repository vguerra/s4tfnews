S4TF News - Issue 20, July 28th 2019

Welcome to the Issue #20 of Swift for Tensorflow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for Tensorflow project.

Enjoy!

News and Articles

  • The 4th release candidate of Swift for Tensorflow v0.4 is now available. Feel free to try it out in Colab or download a pre-built pacakge from here.

  • On friday 26th of July, the 4th SIG Swift Design meeting took place. Here you have the highlights:

    • Anthony Platanios presented the work that has been done on Sequential and Embedding Layers.

    • Brad Larson talked about the plans on refactoring swift-models repository. Find the overview of the plan here.

    • Parker Schuh leaded the discussion on C++ Interoperability and the progress that has been done on the topic. He shared an example showcasing how to enable C++ Interoperability ( see below in the mailing list section).

    You can find the meeting notes and all questions raised in this google doc.

  • Julien Chaumond shared on twitter the port, he and Lysandre did, of OpenAI's GPT-2 to run on-device (using Swift and CoreML, not S4TF but still cool). Check out the Github repo.

The mailing list

  • Anthony Platanios. announced a nice list of packages he's been working on for Reinforcement Learning using Swift for Tensorflow:

    • swift-rl: Reinforcement Learning library with support for environments (e.g. OpenAI Gym) and RL Algorithms.
    • swift-ale: Interface to the Atari Learning Environment.
    • swift-retro: Interface to Gym Retro providing a swift-rl environment for Atari, Sega, etc.

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-26373: Moves utilities to check for memory leaks from tests in AutoDiff/leakchecking.swift to DifferentiableUnittests.

  • PR-26355: Fixes all memory leaks in differentiation of control flow. Resolves TF-552.

  • PR-26345: Fixes two potential issues: usage of freed memory and a memory leak.

  • PR-26313: Fixes unexpected compilation error when differentiating protocol requirements with multiple @differentiable attributes. Resolves TF-676.

  • PR-26050: Enables differentiation of class's methods. Reslves (TF-631).

  • PR-26292: Fixes compiler crash when generating param subset thunk for a partially-applied function. Adds as well diagnostics for differentiation of direct method references. TF-675..

Swift TensorFlow APIs

  • PR-396: Improves messages on deprecated funtions by adding migration instructions.

  • PR-391: Enables usage of control flow AD in Layers: BatchNorm and Dropout. Introduces as well utility withTensorLeakChecking { ... } for checking memory leaks. Resolves #389.

  • PR-346: Introduces the Module protocol, which is equivalent to Layer before this PR but with no contraints on Input. Layer protocol now conforms to newly introduced Module. As well, Embedding now conforms to Module and Sequential has been adapted to work with both Modules and Layers ( with the former restricted to appear only at the beginning of the chain).

  • PR-356: Introduces the reduction parameter for Losses which is a Reduction to be applied on the element-wise loss values. As well, makes some instances of Tensor.max() and Tensor.min() @differentiable.

  • PR-387: Makes useLazyTensor a Thread-local state.

  • PR-384: Uses .moments(:) to compute the mean and variance within LayerNorm and BatchNorm Layers. As well, adds tests for both of them.

  • PR-381: Uses LazyTensor explicitly to extract traces.

  • PR-377: Introduces experimental support for installation of Tensorflow into a Swift toolchain.

  • PR-380: Adds tests for Dense Layer.

Swift Models

  • PR-181: Adds a non-convolutional GAN example on MNIST.

  • PR-179: Replaces Autoencoder's activation function with sigmoid. Resolves #179.

  • PR-178: Makes usage of Swift (instead of Python) for downloading and parsing of CIFAR10 dataset.

Swift for Tensorflow Docs

  • fe3dfd72: Updates notebook install_latest_swift.ipynb so that artifacts are fetched from https://storage.googleapis.com/swift-tensorflow-artifacts/nightlies/latest/....

  • PR-217: Updates instructions to use Swift for Tensorflow toolchain within Xcode.
