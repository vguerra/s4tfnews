S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-28505: Adds differentiability_witness_function instruction assertions.

  • PR-28451: Makes the differentiation pass look up witnesses and make differentiable_witness_function instructions, instead of looking up attributes and making function_refs.

  • PR-24068: Makes Dictionary's KeyPathIterable conformance use indices.

  • PR-28494: Fixes unsilenceable @noDerivative warning. Resolves TF-1018.

  • PR-28492: Fixes differentiation of tuples with single differentiable element. Resolves TF-964.

  • PR-28490: Fixes compiler crash when doing @differentiable attribute type-checking. Resolves TF-1017.

  • 28488: Renames @transposing to @transpose(of:). Resolves TF-992.

  • PR-28480: Adds optimization pass that devirtualizes differentiability witnesses. Resolves TF-919 and TF-994.

  • PR-28483: Bans local-scope derivative registration and adds respective diagnostics. Resolves SR-11849 and TF-881.

  • PR-28416: Adds Differentiable.zeroTangentVectorInitializer.

  • 28477: Fixes aggregate adjoint value accumulation bug. Resolves TF-943.

  • PR-28481: Renames @differentiating to @derivative(of:).

  • PR-28468: Extends support of derivative registration for more declaration kinds: computed properties, subscripts, and initializers. Resolves TF-281.

  • PR-28470: Updates differentiableFunction(from:) to use differentiable function constructor builtins. Resolves SR-11847.

  • PR-28471: Makes transpose(of:) available. Resolves SR-11845.

  • PR-28472: Implies AdditiveArithmetic generic constraints from @differentiable(linear) parameters or results.

  • PR-28469: Adds Builtin.applyTranspose*. Resolves SR-11844 and SR-11851.

  • PR-28466: Cleans up differentiation-related attributes.

  • PR-28467: Adds builtin differentiable/linear function consturctors. Resolves SR-11846.

  • PR-28463: Fixes differentiability witness SIL serialization.
