S4TF News - Issue 1, March 17th 2019

Welcome to the Issue #1 of Swift for Tensorflow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for Tensorflow project.

Enjoy!

News and Articles

  • Prannay Khosla shared on a Medium a post his view on what features a language suited for Machine Learning systems development would need to offer.

  • Back in January 2019, Jeremy Howard wrote a very nice article on which he explores numeric programming in Swift, the article is called High Performance Numeric Programming with Swift: Explorations and Reflections. I encourage you to take some time to read it. The section that I found most interestig is The Warts, where he goes over some pain points he faced when using Swift and highlights areas of improvement for the Swift ecosystem.

  • Richard Wei gave a talk about Swift for Tensorflow back in November 30th 2018 at the O'Reilly AI Conference in San Francisco. You can find the recording of his talk in the Tensorflow's youtube channel.

The mailing list

  • Dan Zheng shared the updated list of thecnical documents for Swift for Tensorflow. If you want to deep dive into the details of the project, you defenitely need to go through the list here.

  • Dan Zheng put together a useful gist detailing how you can select a given Python version for Swift's Python interoperability. Documentation covering this will be added soon but you can use the gist in the meantime ;). See the original message

  • Brad Larson raised a interesting discussion regarding import/exporting models into/from Swift for TensorFlow.
