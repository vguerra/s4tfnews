S4TF News - Issue 52, March 8th 2020

Welcome to the Issue #52 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • This Issue marks one year of Swift for TensorFlow Newsletter. Thank you for your readings and support.

  • The 1st release candidate for Swift for TensorFlow 0.8.0 is now available. Grab a copy from the Release Candidate's section from the installation instructions. The S4TF v0.8 Release Notes document guides you through all changes since v0.7. Please give this release a try and report any issues you find on JIRA or GitHub issues on Swift-APIs.

  • On Friday 6th of March, the 25th SIG Swift Design meeting took place. Here you have the highlights:

    • Dan Zheng and Marc Rasi gave an update on recent developments on Automatic Differentiation capabilities in the Swift Compiler. As well, they gave a live demo, showcasing Differentiable Swift on a physics example. You can find the slides for the presentation here.

    The notes and all questions raised in this google doc.

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-30257: Fixes crash when emitting @noDerivative warnings on property wrapper backing. Resolves TF-1190.

  • PR-30205: Creates cascading symlinks for the TensorFlow library as well on Darwin.

  • PR-30178: Enables @differentiable attribute for setters. Resolves TF-1166.

Swift TensorFlow APIs

  • PR-717: Constraints randomCategorialLogits to be TensorFlowFloatingPoint.

  • PR-718: Reduces uses of Int32 Tensor.

  • PR-706: Adds resize functions based on tf.image.resize. Partially resolves Issue-670.

  • PR-716: Marks Dense.init(weight:bias:activation) as @differentiable.

  • PR-702: Replaces SequentialTests with SequencedTests.

  • PR-709: Introduces reversing of Tensors.

  • PR-707: Avoids in Math.swift reference to name of the module or usege of Tensor<Int32>() for shapes.

Swift Models

  • PR-361: Fixes in Transformer merge pair.

  • PR-356: Teaches CheckpointIndexWriter to write TensorFlow 1.x/2.x compatible checkpoints.

  • PR-337: Adds KuzushijiMNIST dataset, a MNIST wrapper and a Autoencoder2D demo.

  • PR-340: Introduces an implementation of a TensorFlow v2 checkpoint writer.

  • PR-327: Improves Image flexibility.

  • PR-349: Makes normalization of CIFAR10 images optional.

  • PR-353: Updates documentation on how to test with CMake.
