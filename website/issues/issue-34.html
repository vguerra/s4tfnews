S4TF News - Issue 34, November 3rd 2019

Welcome to the Issue #34 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • Jacopo Mangiavacchi has shared his SwiftTFRecords library, which facilitates producing and importing data in the TFRecords format directly in Swift.

  • The design document for SwiftRT, a high-peformance Tensor runtime layer presented by Edward Connell during last SIG Swift Design meeting, is now available .

  • Paige Bailey and Brennan Saeta were present at TensorFlow World 2019 the past week to talk about Swift for TensorFlow - From the Foundations. Check out the recording, which is available on YouTube.

  • As well, during TensorFlow World 2019, Mars Geldard, Tim Nugent, and Paris Buttfield-Addison ran a tutorial titled Swift for TensorFlow in 3 hours . You can find the material used for this tutorial on GitHub.

  • Nightly development snapshopts of Swift for TensorFlow are now available for Ubuntu 18.04 (CUDA 10.1). You can find then in the Development Snapshots section of the install instructions.

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-27971: Makes Array.DifferentiableView unconstrained which allows to re-enable ParseableInterface tests. Resolves TF-489.

  • PR-27985: Makes sure to store original declaration in DifferentiableAttr to ease resolution of parameter indices.

  • PR-27988: Cleans up the type-checking for @transposing attribute.

  • PR-27923: Enables LoadableByAddress IRGen transformation.

  • PR-27919: Makes sure that DifferentiabilityWitnessFunctionInst stores a pointer to the witness.

  • PR-27918: Renames TransposingAttr::getParameterIndexSubset to TransposingAttr::getParameterIndices.

Swift TensorFlow APIs

  • PR-538: Adds constraint Element: Differentiable to Array.DifferentiableView exntentions.
