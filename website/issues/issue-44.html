S4TF News - Issue 44, January 12th 2020

Welcome to the Issue #44 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

News and Articles

  • Jacopo Mangiavacchi wrote an article called: Swift loves TensorFlow and CoreML. He explains how to export from S4TF to CoreML and enable on device model personalization / re-training. All done in Swift, no Python code involved.

  • The SIG Swift Design Meeting Notes has been updated with links to all missing recordings from previous sessions.

  • On Friday 10th of January, the 20th SIG Swift Design meeting took place. Here you have the highlights:

    • Brennan Saetatalked about Penguin which is a toolkit of abstractions for parallel programming. You can find the slides of the presentation here.

    You can find the meeting recording, notes and all questions raised in this google doc.

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-29137: Adds cross-file registration flag to .swiftinterface. Resolves TF-1101.

  • PR-29134: Moves all stdlib derivatives to separate files.

  • PR-28616: Simplifies SILGen thunking and set correct thunk linkage.

  • PR-29066: Fixes on KeyPathIterable.

  • PR-29046: Uses size_t for the indexing type.

  • PR-29030: Uses @derivative(of:) for array and floating point derivatives.

  • PR-28950: Fixes @transpose attribute type-checking. Resolves TF-1060 and TF-997.

  • PR-28933: Use @derivative attribute to register tgmath derivatives.

  • PR-29016: Enables derivative registration for imported C functions. Resolves TF-1087.

  • PR-29015: Makes the [parameters ...] argument always required in the textual representation for {differentiable,linear}_function instructions.

Swift TensorFlow APIs

  • PR-616, PR-610: Uses explicitly Float variant of math APIs. This allows Tensor to build on Windows.

  • PR-611: Corrects a thread allocation leak.

  • PR-612: Adds He and LeCun initializers.

  • PR-605: Adds preconditions for moments, logSumExp and standardDeviation. Partially resolves Issue-517.

  • PR-578: Adds Add eye and svd functions.

  • PR-608: Handles C imports for Windows.

Swift Models

  • PR-274: Update FastStyleTransfer example with new checkpoint loader and image functions. Partially resolves Issue-225

  • PR-273: Reworks MiniGo and Transformer models to use a downloading checkpoint reader. Partially resolves Issue-225.

  • PR-272: Restructures dataset downloading code.
