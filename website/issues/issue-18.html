S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-: Fixes compiler bug due to VJPEmitter's SubstitutionMap not using the VJP's generic signature.

  • PR-26113: Prevents to specify jvp: or vjp for @differentiable attribute on protocl requirement.

  • PR-25974: Allows conditional differentiability on Protocol requirements.

  • PR-26023: Introduces @differentiable versions of Array.map and Array.reduce called Arrray.differentiableMap and differentiableReduce respectively.

  • PR-25954: Introduces a JVPEmitter.

  • PR-25967: Fixes memory leaks during differentiation.

Swift TensorFlow APIs

  • PR-360: Adds @differentiable attribute for overloads of min(_:_:) and max(_:_:).

  • Introduction of new the following operations for Tensors:

    • PR-358: Adds cumulativeProduct(alongAxis:exclusive:reverse:).
    • PR-353: Adds cumulativeSum(alongAxis:exclusive:reverse:).

  • PR-354, PR-355: Introuces support for Tensor moments. As consequence a custom VJP for batch normalization is not needed.

  • PR-350: Fixes bug on computation of rsqrt's gradient (_vjpRsqrt).

  • PR-347: Makes Array and Array.DifferentiableView conform to aggregate math protocols and collection protocols when appropiate.

  • PR-341: Fixes bug on computation of sigmoid's gradient (_vjpSigmoid).

  • PR-340: Introduces typealias TensorflowSeed to standarize usage of seeds.

  • PR-339: Adds sha1 hashing algorithm as an extention of Array<Int8>.

  • PR-336: Preconditions initializer init(arrayLiteral:) to reject initializations with no elements. This applies to ShapedArraySlice, ShapedArray and Tensor.

  • PR-334: Bug fix on initialization of TransposedConv2D Layer.
