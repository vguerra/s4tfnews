S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-25715: Adds diagnostics on differentiable functions with inout parameters.

  • PR-25868: Fixes bug on AST serialization on a differentiated function that has control flow.

  • PR-25861: Fixes bug on type-checking for @differentiable(wrt: self) so that Protocols conforming to Differentiable can define @differentiable computed properties.

  • PR-25855: Renames Differentiable.withGradient to Differentiable.withDerivative.

  • PR-25851: Fixes bug in JVP/VJP method thunking in cross-module setup.

  • PR-25804: Adds mangling for function types: @differentiable, escaping @diffferentiable, @differentiable(linear) and escaping @differentiable(linear).

  • PR-25803: Partially fixes compiler crash in @differentiable function thunking.

  • PR-25772: Introduces PointwiseMultiplicative protocol.

  • PR-25778: Fixes building with Xcode (-x or --xcode).

  • PR-25724: Fixes a SIL verification bug when calling valueWithPullback inside a method of a generic class.

  • PR-25708: Fixes temporarly error undeclared type '__lldb_expr_' that you might get in REPL or when running on Jupyter.

  • PR-25706: Changes preset tensorflow_osx_base to use option release instead of release-debuginfo. This unlocks producing a Swift for Tensorflow toolchain under macOS.

  • PR-25699: Fixes compiler crash during creation of subset-parameters thunk.

  • PR-25691: Introduces top-level function withoutDerivative(at:) and removes Differentiable.withoutDerivative().

Swift TensorFlow APIs

  • PR-318: Introduces helper function randomSeedForTensorFlow().

  • PR-317: Generalizes TransposedConv2D over Scalar type.

  • PR-316: Fixes unexpected behavior in matmul and it's VJP due to reshaping of arguments.

  • Added Layers:

    • PR-316: Adds ZeroPadding1D, ZeroPadding2D, and ZeroPadding3D.

    • PR-298: Adds Function layer.

  • On Optimizers:

    • PR-311: Removes RiemannSGD.

    • PR-314: Introduces AMSGrad optimizer.

    • PR-304: Introduces AdaMax optimizer.

    • PR-302: Introduces AdaDelta optimizer.

  • PR-301: Allows manually promoting constants to inputs in tracing for LazyTensors.

  • PR-218: Simplifies optimizers by using generalized vector math.

  • PR-296: Triggers immediate execution of Lazy Tensor operations that have no output.

  • PR-297: Makes Tensor conform to PointwiseMultiplicative protocol.

  • PR-290: Extends TensorArrayProtocol with default implementations for _tensorHandles property and initialization with a collection of _AnyTensorHandles.

  • PR-293: Turns off lazy evaluation after LazyTensor test have run to avoid runtime failures for tests executed after.

  • PR-266: Adds ability to convert a LazyTensorTrace into a TFFunction.

  • PR-295, PR-292: Substitutes assert(false, msg) by faltaError(msg) as the compiler removes the former when building in release mode.

  • Adding default values for some operator's parameters:

    • PR-287: strides and padding parameters for convolution operators.
    • PR-286: dilatations parameter for 2D convolution operators.

  • PR-147: Adds log1mexp(:) which implements log(1 - exp(x)).

  • PR-285: Substitutes usage of Swift.withoutDerivative(at:in:) by withoutDerivative(at:).

Swift Jupyter

  • PR-67: Enables display of images ( PNGs ) in Jupyter using pure Swift.
