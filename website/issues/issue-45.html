S4TF News - Issue 45, January 19th 2020

Welcome to the Issue #45 of Swift for TensorFlow Newsletter. A newsletter letter that summarizes weekly activity around the Swift for TensorFlow project.

Enjoy!

S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-29225: Removes the unnecessary code in CTensorFlow.

  • PR-29290: Adds JVPs for tgmath functions except remainder(_:_:) and fmod(_:_:). Partially resolves TF-1108.

  • PR-29240: Defines deerivatives Float80 only for supported platforms.

Swift TensorFlow APIs

  • PR-636: Updates Dockerfile to add CMake based CI build.

  • PR-631: Removes cancellation point handling.

  • PR-624: Abstracts Mutex interfaces.

  • PR-627: Adjusts flags for inlining and partial specializations.

  • PR-623: Abstracts TLS handling.

  • PR-622: Abstracts Thread interfaces.

  • PR-604: Adds linear algebra operations det and slogdet.

  • PR-621: Generate Raw ops from Tensorflow2.1.0rc1. Resolves Issue-619.

  • PR-614: Extends preconditions. Partially resolves Issue-517.

  • PR-617: Adds a CMake based build.

  • PR-595: Extends tests in MathTests.swift.

  • PR-615. Makes TensorShape conform to Collection-like protocols. Resolves Issue-609.

Swift Models

  • PR-275: Makes ResNet models a little more flexible so that they generalize better to other datasets.
