S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-28787: Moves thunk generation utilities to Utils/Differentiation/Thunk.h.

  • PR-28782: Adds @transpose attribute serialization/deserialization.

  • PR-28772: Fixes derivative generic signature calculation when same-type requirements bind all generic parameters to concrete types. Resolves TF-1059 and .

  • PR-28761: Fixes syntax support for @derivative and @transpose. Resolves TF-1009.

  • PR-28735: Puts the generic signature of the original function in the DifferentiableAttribute. Resolves TF-1046.

  • PR-28716: Forbids overly-constrained derivative requirement implementations.

  • PR-28717: Changes @derivative and @transposing type-checking to not reuse @differentiable diagnostics. Rename shared diagnostics to have a general name. Resolves TF-998 and TF-1051.

  • PR-28699: Mangles derivative generic signatures for derivative functions. Resolves TF-680 and TF-1037.

  • PR-28693: Adds Array.zeroTangentVectorInitializer, a closure that produces a TangentVector of zeros with the same count as self.

  • PR-28623: Adds support in LoadableByAddress for when the differentiable function conversion instructions are operating on generic functions. Resolves TF-961.

  • PR-28662: Makes sure that cloned curry thunks generated by differentiation create a differentiable_function instruction before any dealloc_stack instructions to prevent alloc_stack/dealloc_stack ordering issues.. Resolves TF-1039.

  • PR-28621: Lowers directly @derivative attributes to differentiability witnesses during SILGen. Resolves TF-835.

Swift TensorFlow APIs

  • PR-585: Adds arity-3 Tensor differential operators. Resolves Issue-344.

  • PR-582: Adds a stub which represents the device of a tensor.
