S4TF Commits

Swift Compiler (tensorflow branch)

  • PR-31249: Enable cross-file derivative registration by default on. Resolves TF-1097.

  • PR-31216: Adds --no-tensorflow-swift-apis option to utils/build-toolchain-tensorflow to have a tensorflow toolchains without tensorflow-swift-apis library.

  • PR-31158: Removes TensorFlow workarounds in build logic.

Swift TensorFlow APIs

  • PR-882: Avoids unnecessary Tensor materialization in Tensor-Scalar op derivatives.

  • PR-881: Makes use of Device more explicit.

  • PR-765: Introduces GaussianDropout. Partially resolves Issue-685.

  • PR-878: Allows picking the best device automatically.

  • PR-857: Updates to CUDA 10.2.

  • PR-851: Builds TensorFlow with x10 support if building an x10 toolchain.

  • PR-863: Introduces various fixes to allow building with the X10 backend on Windows.

  • PR-853: Makes sure to preserve the source device in the output device when falling back to TFEager.

  • PR-862: Improves x10's build for Windows, it reuses the existing ThreadLocalStorage type in swift-apis for TLS access to mask out differences between platforms.

  • PR-860: Deprecate Layer.call(_:).

  • PR-854: Restructures the X10 swift bindings to split X10 and CTensorFlow.

  • PR-847: Makes sure that Tensors be on consistent device for bandapart operations.

Swift Models

  • PR-365https://github.com/tensorflow/swift-models/pull/365: Introduces pix2pix example.

  • PR-446: Adds support for COCO dataset.

  • PR-444: Restructures benchmarks to be batch-based in training.

  • PR-453: Makes GPT2-WikiText2 train on TPU with x10.
